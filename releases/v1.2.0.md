# SmartAgri V1.2.0 — MQTT Remote Architecture

**状态：预发布 / 需要实际硬件联调**

V1.2 将 V1.1 中“Flask 直接占用 CH340 串口”的模式拆分为本地 Gateway 与远端服务：MCU 串口协议保持兼容，Gateway 通过 TLS MQTT 与 EMQX 通信，远端 Flask/PostgreSQL 保存数据并对外提供审计化控制 API。

## 新增内容

### Local Gateway

- 新增 `dashboard/gateway/` 独立本地程序。
- Gateway 是唯一 MCU CH340 串口拥有者。
- 复用 Protocol 2 的 `DATA`、`ACK`、`ERR`、`GET`、`SET` 与 `CTRL`。
- MQTT 主题：遥测、最新状态、在线状态、控制命令、命令结果。
- SQLite WAL 离线队列：网络故障下持久化上报与控制结果。
- QoS 1 重投去重：重复 `request_id` 不会重复执行 MCU 控制。
- MQTT TLS 校验为强制项，拒绝明文 MQTT 配置。

### Remote Service

- 新增 Flask 远端 API、PostgreSQL 数据模型和独立 MQTT Worker。
- 持久化设备、最新状态、遥测历史、命令生命周期和操作者记录。
- 提供 viewer/operator 角色模型；控制 API 返回 `202 Accepted`，等待 Gateway 的 MCU ACK/ERR 结果。
- Docker Compose 部署：Flask/Gunicorn、MQTT worker、PostgreSQL。

### EMQX

- 使用固定 EMQX `5.8.7`。
- MQTT 使用 TLS `8883`，关闭公网明文 MQTT 和 WebSocket 暴露。
- 每个 Gateway/后端使用独立 MQTT 账号。
- 默认拒绝 ACL；Gateway 仅能访问自己设备的 MQTT 主题。

## 兼容性

- MCU 固件与 Protocol 2 首个 V1.2 版本不需要变更。
- 活动硬件仍是 STM32F103C8T6；`Legacy_C51/` 不是当前部署目标。
- ESP-01S HTTP 上传在 V1.2 默认停用；Gateway 是唯一云同步路径。

## 部署前检查

1. 为实际 Gateway 配置正确的 `COM` 端口。
2. 将 EMQX CA 证书复制到 Gateway 主机并设置 `SMARTAGRI_GATEWAY_MQTT_CA_PATH`。
3. 设置唯一 Gateway MQTT 凭据；不得使用 Dashboard 账号。
4. 在远端服务首次启动后创建首个 operator 账号。
5. 先执行非破坏性阈值操作确认 MCU `ACK`，再测试有 TTL 限制的执行器控制。
